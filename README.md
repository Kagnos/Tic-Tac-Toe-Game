# Tic-Tac-Toe-Game

I learned a couple important things from creating this game and I grew a lot of my skills. On this project I worked with and got better at:

* Flexbox and grid
* Media queries
* Arrays
* Writing pseudocode
* Splitting large functions into smaller ones
* forEach and other methods
* Node lists
* Event listeners
* DOM manipulation
* Objects
* Data attributes
* IIFEs
* Factory functions

One of the most important things I learned and implemented on this project is using one piece of data to dictate the DOM instead of referencing the DOM. I used an array to keep track of the game board and I called the function updateBoard() to update the DOM to match my data. This way if the DOM ever messes up it doesn't matter because it isn't a point of authority anymore. I'm not checking the DOM for game over, I'm checking my data and updating the DOM to reflect it.

The other important thing I learned from working through this project is how crucial it is to work on the logic before the styling. In past projects I have updated the CSS to always match and grow with the logic, but this has required me to constantly switch between logic and styling, always keeping the two up to date and in sync with each other. In this project I wrote the framework in pseudocode, implemented the logic, and left the styling till the end. As a result it took me less than half the time it usually takes me to complete a project.

In conclusion, this project has taught me a lot and helped me understand crucial ideas. This has been one of my best projects to date, but I know I can build bigger, better things. I will keep learning, building, and trying my best.
